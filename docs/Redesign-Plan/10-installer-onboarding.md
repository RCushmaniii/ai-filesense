# Installer and Onboarding Flow

> **Document:** 10-installer-onboarding.md  
> **Purpose:** Define first-run experience, installation, and permissions requests

---

## Overview

The installer and onboarding flow must accomplish:

1. **Install cleanly** â€” No bloatware, no surprises
2. **Request permissions gracefully** â€” Explain why, ask once
3. **Deliver value immediately** â€” First organization within 5 minutes of install
4. **Build trust early** â€” Transparency about what the app does

---

## Installation Flow

### Distribution Options

| Method | Target User | Notes |
|--------|-------------|-------|
| `.exe` installer | Most users | Standard Windows installer |
| `.msix` package | Microsoft Store | Sandboxed, auto-update |
| Portable `.zip` | Power users | No install required |

## First-Run Onboarding

After installation, the app launches into onboarding. This is separate from the main user journey (Screens 1-9) and only appears once.

### Onboarding Goals

| Goal | How |
|------|-----|
| Build trust | Explain privacy, local-only processing |
| Set expectations | What the app does and doesn't do |
| Request permissions | Folder access, optional integrations |
| Personalize (optional) | Quick context for better results |

## Summary

| Phase | Screens | Goal |
|-------|---------|------|
| Installation | I-1 to I-4 | Clean install, minimal friction |
| Onboarding | O-1 to O-6 | Build trust, request permissions |

- Permissions requested with clear explanations
- Value delivered within 5 minutes of install
- Tour is optional, never blocking
