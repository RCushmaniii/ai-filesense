# Settings Screen Specifications

> **Document:** 11-settings-specifications.md  
> **Purpose:** Define all user-configurable options and their UI

---

## Overview

Settings are organized into logical groups with progressive disclosure. Most users will never need to change defaults, but power users have full control.

### Design Principles

1. **Sensible defaults** â€” Works great out of the box
2. **Progressive disclosure** â€” Simple view first, advanced hidden
3. **Immediate feedback** â€” Changes apply instantly where safe
4. **Easy reset** â€” "Restore defaults" always available
5. **Context help** â€” Every setting has explanation

  "export_version": "1.0",

  "app_version": "2.1.0",

    "notifications": { ... },
    "privacy": { ... },
    "advanced": { ... }
  }
}

### Import Validation

```python
def validate_settings_import(data: dict) -> ValidationResult:
    """Validate imported settings file."""
    
    errors = []
    warnings = []
    
    # Check version compatibility
    if data.get('export_version') != '1.0':
        warnings.append("Settings from different versionâ€”some may not apply")
    
    # Validate each section
    for section in ['general', 'scanning', 'organization']:
        if section not in data.get('settings', {}):
            errors.append(f"Missing section: {section}")
    
    # Validate paths exist
    for folder in data.get('settings', {}).get('scanning', {}).get('default_locations', []):
        path = os.path.expandvars(folder.get('path', ''))
        if not os.path.exists(path):
            warnings.append(f"Folder not found: {path}")
    
    return ValidationResult(
        valid=len(errors) == 0,
        errors=errors,
        warnings=warnings
    )
```
